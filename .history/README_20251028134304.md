# ECSS Glossaire — application locale

Petit projet local pour rechercher rapidement des termes et abréviations de la norme ECSS.

Pré-requis
- Python 3.8+

Étapes rapides
1. Générer le JSON à partir du fichier texte (fourni) :

```bash
python3 tools/parse_glossary.py ecss_glossaire.txt
```

Le script écrit `data/glossary.json`.


2. Servir le dossier pour ouvrir l'interface web :

Option A — servir la racine du projet (recommandé) :

```bash
cd /Users/aldricparent/Documents/Spatial/ECSS
python3 -m http.server 8000
# puis ouvrir http://localhost:8000/web/
```

Cette option permet au frontend (`web/index.html`) de charger `data/glossary.json` situé à la racine du projet.

Option B — servir seulement le dossier `web` :

```bash
cd /Users/aldricparent/Documents/Spatial/ECSS/web
python3 -m http.server 8000
# puis ouvrir http://localhost:8000
```

Si vous servez uniquement `web/`, copiez `data/glossary.json` dans `web/data/` :

```bash
cp ../data/glossary.json web/data/glossary.json
```

3. Rechercher un terme dans la page. Le champ propose la recherche sur terme et définition.

Fichiers ajoutés
- `tools/parse_glossary.py`: parseur CSV -> `data/glossary.json`.
- `web/` : frontend minimal (`index.html`, `app.js`, `styles.css`).
- `data/glossary.json` : généré par le parseur (non inclus par défaut).

Notes
- Ouvrir `web/index.html` directement via le système de fichiers n'affichera pas `data/glossary.json` pour des raisons de sécurité du navigateur ; servez le dossier via un serveur HTTP comme indiqué.
